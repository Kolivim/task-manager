## Стек технологий

Kotlin 1.9.23, Java 17

Spring Boot 3.2.4, Spring WebFlux, Project Reactor

Spring JDBC JdbcClient, native SQL, Flyway

База данных	H2

JUnit 5, Mockito, StepVerifier, WebTestClient

Gradle (Kotlin DSL)

SLF4J


## Структура проекта

src/main/kotlin/
├── controller/          # REST-контроллеры
├── service/             # Бизнес-логика (Mono/Flux)
├── repository/          # Доступ к данным (JdbcClient, native SQL)
├── model/               # Сущность Task и Enum TaskStatus
├── dto/                 # DTO для запросов/ответов
├── exception/           # Кастомные исключения и глобальный обработчик
└── config/              # Конфигурационные классы (при необходимости)

src/main/resources/
├── application.yml      # Конфигурация приложения
├── db/migration/        # SQL-скрипты миграций Flyway

Главный класс приложения: TaskManagerApplication.kt 
(расположен в корне src/main/kotlin, без явного пакета, но с указанием scanBasePackages).


## Быстрый старт

Требования
JDK 17 или выше
Gradle (обёртка gradlew уже включена)


## Для запуска выполните следующие шаги:

1. Клонируйте репозиторий

2. Выполните сборку при помощи команды bash:
./gradlew clean build

3. Запустите приложение при помощи команды bash:
./gradlew bootRun

- Или через IDE, запустив main в TaskManagerApplication.kt.

Приложение стартует на порту 8080.
База данных H2 по умолчанию работает в файловом режиме (данные сохраняются в папку data/).
Для переключения на in‑memory, раскомментируйте соответствующую строку в application.yml

## API endpoints

Базовый URL: `http://localhost:8080/api/tasks`

### Основной контроллер (TaskController)

| Метод   | Endpoint                     | Описание                                                   |
|---------|------------------------------|------------------------------------------------------------|
| `POST`  | `/`                          | Создать задачу                                             |
| `GET`   | `/{id}`                      | Получить задачу по ID                                      |
| `GET`   | `?page=0&size=10&status=NEW` | Получить список задач (пагинация + фильтрация, по статусу) |
| `PATCH` | `/{id}/status`               | Обновить статус задачи с переданным ID                     |
| `DELETE`| `/{id}`                      | Удалить задачу по ID                                       |


## Конфигурация

Основные настройки находятся в `application.yml`:

| Параметр                | Описание                                                |
|-------------------------|---------------------------------------------------------|
| `spring.datasource.url` | Подключение к БД H2                                     |
| `flyway.enabled`        | Активация миграций                                      |
| `logging.level.*`       | Настройка уровней логирования для различных компонентов |
| `logging.file.name`     | Имя файла для сохранения логов                          |

### 📝 Примечания

Файловая H2 (jdbc:h2:file:./data/taskdb) сохраняет данные на диск в папку data. 
При повторном запуске база восстанавливается.

Для переключения между доступными режимами H2 достаточно закомментировать/раскомментировать соответствующую строку url.

Логи пишутся помимо консоли в файл log папки logs.

## Схема базы данных

### `tasks` — таблица задач

| Поле          | Тип                     | Ограничения                             | Описание                                                        |
|---------------|-------------------------|-----------------------------------------|-----------------------------------------------------------------|
| `id`          | `BIGINT`(autoIncrement) | `PRIMARY KEY`                           | **Идентификатор** задачи                                        |
| `title`       | `VARCHAR(100)`          | `NOT NULL`                              | Заголовок задачи (обязательный, длина от 3 до 100 символов)     |
| `description` | `VARCHAR(500)`          | `NULL`                                  | Описание задачи (необязательное поле)                           |
| `status`      | `VARCHAR(20)`           | `NOT NULL`                              | Текущий статус задачи (enum: NEW, IN_PROGRESS, DONE, CANCELLED) |
| `created_at`  | `TIMESTAMP`             | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Дата и время создания задачи                                    |
| `updated_at`  | `TIMESTAMP  `           | `NOT NULL`, `DEFAULT CURRENT_TIMESTAMP` | Дата и время последнего обновления задачи                       |

**Индексы:**
- idx_tasks_created_at — для быстрой сортировки по дате создания (по убыванию)
- idx_tasks_status — для ускорения фильтрации по статусу

> **Значения поля `status`:** только `NEW`, `IN_PROGRESS`, `DONE`, `CANCELLED`. 
> Любые другие значения будут отклонены на уровне приложения. 
> Проверка целостности дополнительно реализована в коде через enum TaskStatus
